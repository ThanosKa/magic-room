# Security Policy

## Supported Versions

We take security seriously and actively maintain the following versions with security updates:

| Version | Supported          |
| ------- | ------------------ |
| main    | :white_check_mark: |
| < 1.0  | :x:                |

## Reporting a Vulnerability

If you discover a security vulnerability in Magic Room, please help us by reporting it responsibly.

**Please do not report security vulnerabilities through public GitHub issues.**

Instead, please report security vulnerabilities by emailing:
**kazakis.th@gmail.com**

### What to Include

When reporting a vulnerability, please include:

- A clear description of the vulnerability
- Steps to reproduce the issue
- Potential impact and severity
- Any suggested fixes or mitigations (optional)
- Your contact information for follow-up

### Our Commitment

We commit to:

- **Acknowledge** your report within 3 business days
- **Investigate** and assess the vulnerability within 7 business days
- **Keep you informed** throughout the process
- **Credit you** appropriately once the issue is resolved (if desired)

## Response Process

1. **Initial Response**: We'll acknowledge receipt of your report within 3 business days
2. **Investigation**: Our security team will investigate and assess the vulnerability
3. **Updates**: We'll provide regular updates on our progress
4. **Fix Development**: If confirmed, we'll develop and test a fix
5. **Disclosure**: We'll coordinate disclosure with you
6. **Resolution**: The fix will be deployed and announced

## Scope

This security policy applies to:

- **Application Code**: The Magic Room web application codebase
- **Infrastructure**: Defined cloud infrastructure and deployment configurations

### Out of Scope

This policy does not apply to:

- **Third-party Services**: Issues in external services (Supabase, Clerk, Stripe, OpenRouter, etc.)
- **User Data**: Vulnerabilities requiring access to user accounts or data
- **Denial of Service**: Basic DoS attacks that don't reveal sensitive information

## Safe Harbor

We consider security research conducted in accordance with this policy to be authorized. We won't initiate legal action against researchers who follow these guidelines.

## Recognition

We appreciate security researchers who help keep Magic Room safe. With your permission, we'll acknowledge your contribution in our security advisories and project documentation.

## Questions

If you have questions about this policy or the reporting process, please email kazakis.th@gmail.com.
